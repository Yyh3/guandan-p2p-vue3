# guandan-p2p-vue3 UI / 美工优化修改文档

## 1. 文档说明

本文档基于最新版项目的实际运行结果，对以下页面进行了 UI 与体验审查：

- 首页
- 加入房间页
- 房间大厅
- AI 对局配置页
- 正式牌桌
- 移动端牌桌预览页
- 设置页
- 战绩页
- 规则页
- 结算页

审查尺寸：

- 移动端：`390 × 844`
- 桌面端：`1440 × 1000`

本文档不仅包含视觉建议，也包含会直接影响页面展示的布局、运行和交互问题。

---

# 2. 总体评价

当前项目已经形成较明确的棋牌游戏视觉风格：

- 深蓝 / 蓝紫桌布
- 暖金色装饰
- 木质桌边
- 半透明玻璃面板
- 扑克牌元素
- 发光描边与渐变按钮

项目已不属于“无设计的功能 Demo”，但仍存在以下主要问题：

1. 首页、房间大厅和正式牌桌的视觉语言不统一；
2. 金色、玻璃拟态、发光描边使用过量；
3. 页面信息层级不清晰；
4. 原生 Emoji 与整体商业棋牌游戏风格不统一；
5. 房间大厅信息拥挤；
6. 移动端存在横向溢出和文字挤压；
7. 设置页存在运行错误；
8. 小字号和低透明度文字过多；
9. 牌桌装饰面积过大，核心操作区反而偏小。

建议本轮 UI 优化遵循：

> 先修阻塞问题，再统一设计系统，最后优化动画和细节。

# 4. 全局视觉系统修改

## 4.1 统一主视觉

建议全项目统一为：

> 深海蓝桌布 + 暖金强调 + 少量木质边缘

### 推荐原则

- 首页、加入房间、设置页：
  - 使用统一深蓝渐变背景；
  - 不绘制大面积木质桌边；
- 房间大厅：
  - 使用与正式牌桌一致的桌布蓝；
- 正式牌局：
  - 保留木质桌边，但减小厚度；
- 结算页：
  - 延续深蓝 + 金色主按钮；
- 棕色：
  - 只用于桌边、木纹或牌桌结构；
  - 不作为整页主背景。

### 建议颜色变量

```css
:root {
  --bg-page: #071426;
  --bg-panel: rgba(16, 38, 66, 0.88);
  --bg-panel-soft: rgba(20, 48, 82, 0.68);

  --table-blue: #0d3d66;
  --table-blue-deep: #082c4b;

  --gold-primary: #f4c45e;
  --gold-hover: #ffd978;
  --gold-soft: rgba(244, 196, 94, 0.22);

  --text-primary: rgba(255, 255, 255, 0.96);
  --text-secondary: rgba(255, 255, 255, 0.82);
  --text-muted: rgba(255, 255, 255, 0.64);

  --danger: #ef6262;
  --success: #58c68b;

  --border-soft: rgba(255, 255, 255, 0.12);
  --border-gold: rgba(244, 196, 94, 0.36);
}
```

## 4.2 减少玻璃拟态和发光效果

当前存在的问题：

- 几乎所有按钮都有金色边框；
- 所有面板都有透明背景和阴影；
- 多个元素同时发光；
- 页面缺乏真正的视觉焦点。

### 建议建立三级操作层级

#### 一级按钮

只用于最重要的操作：

- 开始游戏
- 加入房间
- 出牌
- 重开一局

```css
.btn-primary {
  background: linear-gradient(180deg, #ffd978 0%, #e9ad3f 100%);
  color: #2a1d08;
  border: none;
  box-shadow: 0 8px 24px rgba(233, 173, 63, 0.26);
}
```

#### 二级按钮

用于：

- AI 对战
- 邀请好友
- 提示
- 理牌
- 返回大厅

```css
.btn-secondary {
  background: rgba(255, 255, 255, 0.08);
  color: var(--text-primary);
  border: 1px solid var(--border-soft);
}
```

#### 三级按钮

用于：

- 设置
- 规则
- 复制
- 返回
- 更多

```css
.btn-text {
  background: transparent;
  color: var(--text-secondary);
  border: none;
}
```

## 4.3 统一圆角和阴影

建议统一：

```css
:root {
  --radius-sm: 8px;
  --radius-md: 14px;
  --radius-lg: 20px;

  --shadow-panel: 0 16px 40px rgba(0, 0, 0, 0.24);
  --shadow-button: 0 8px 18px rgba(0, 0, 0, 0.18);
}
```

避免每个组件分别设置不同阴影、不同圆角。

---

## 4.4 替换原生 Emoji

当前项目大量使用：

- 📱
- 🤖
- 📖
- ⚙
- 🀄
- 🔗
- 👑

问题：

- 不同 Android 机型显示差异大；
- 与金属、玻璃、牌桌风格不协调；
- 基线、尺寸、颜色无法统一。

### 修改建议

使用统一 SVG 图标库，例如：

- Lucide
- Heroicons
- 自绘棋牌图标

统一规范：

```css
.icon {
  width: 20px;
  height: 20px;
  stroke-width: 2;
  color: currentColor;
}
```

建议：

- 默认图标：白色或浅灰；
- 激活图标：金色；
- 不要所有图标都带圆圈和发光描边；
- 皇冠、奖杯等强调图标可以保留金色。

# 5. 字体与可读性

当前项目大量使用：

```css
font-size: 10px;
font-size: 11px;
color: rgba(255, 255, 255, 0.5);
```

在真实手机上可读性偏低。

## 推荐字号

| 内容类型 | 推荐字号 |
|---|---:|
| 页面主标题 | 24–28px |
| 卡片标题 | 18–20px |
| 按钮文字 | 16–18px |
| 正文 | 14–16px |
| 辅助文字 | 12–13px |
| 极小徽章 | 不低于 11px |

### 建议变量

```css
:root {
  --font-page-title: 28px;
  --font-section-title: 20px;
  --font-body: 15px;
  --font-button: 17px;
  --font-caption: 13px;
}
```

辅助文字建议：

```css
.text-secondary {
  color: var(--text-secondary);
}

.text-muted {
  color: var(--text-muted);
}
```

不要只靠降低透明度制造层级，应配合：

- 字号
- 字重
- 间距
- 图标
- 分组

# 6. 首页修改

## 当前问题

1. 四个按钮视觉权重完全相同；
2. 用户看不出首选入口；
3. 副标题对比度偏低；
4. 桌面端内容区域偏小，留白过多；
5. 大面积半圆桌边占据页面；
6. Logo 扑克牌图形略像临时 CSS 图形。

## 推荐布局

```text
扑克牌 Logo

掼蛋
无需服务器，同一热点即可开局

[ 开始游戏 ]

[ 加入房间 ]    [ AI 对战 ]

游戏规则

设置                         玩家头像 / 昵称
```

## 修改建议

### 主按钮

“开始游戏”使用唯一实体金色按钮。

### 次级按钮

“加入房间”和“AI 对战”使用半透明深蓝按钮。

### 规则入口

“游戏规则”降级为文字按钮。

### 副标题

将：

```css
.logo-sub {
  color: var(--gold-soft);
}
```

修改为：

```css
.logo-sub {
  color: rgba(255, 255, 255, 0.70);
  font-size: 14px;
  line-height: 1.5;
}
```

### 页面宽度

桌面端建议：

```css
.home-content {
  width: min(460px, calc(100vw - 32px));
}
```

避免内容过窄、周围空白过多。

# 8. 房间大厅修改

房间大厅是当前最需要重新布局的页面。

## 当前问题

顶部同时存在：

- 房间号
- IP 地址
- 二维码
- 复制按钮
- 人数
- 规则
- 网络状态
- 房主头像
- 邀请说明

导致：

- 信息拥挤；
- IP 地址被换行；
- 按钮贴边；
- 房主头像与顶部区域冲突；
- 用户不知道首先应该做什么。

---

## 8.1 重构顶部栏

建议只保留：

```text
房间 123456          1 / 4 人
网络正常
```

布局：

```css
.room-header {
  display: grid;
  grid-template-columns: 1fr auto;
  align-items: center;
  gap: 8px;
}
```

---

## 8.2 二维码改为弹窗或底部抽屉

不要常驻右上角。

点击：

```text
[ 邀请好友 ]
```

后弹出：

```text
邀请好友

[ 二维码 ]

192.168.43.1:8848        [复制]

让好友连接同一热点后扫码加入
```

这样可以释放顶部空间。

---

## 8.3 简化空座位

当前空座位同时出现：

- 虚线圆框
- 数字徽章
- 问号
- 等待加入文字
- 动效

信息重复。

建议仅保留：

```text
浅色头像轮廓
等待加入
```

房主使用头像角标表达：

```text
头像右上角小皇冠
```

不需要同时使用：

- 皇冠
- 金色光圈
- “房主”文字标签

---

## 8.4 强化队友和对手关系

掼蛋中“对家是队友”非常重要。

建议：

- 队友座位使用蓝金色轮廓；
- 对手座位使用红紫色轮廓；
- 中心区域标注：
  - “我的队友”
  - “对手”
- 或使用细线连接队友座位。

---

## 8.5 收起非核心功能

以下功能不应在等待大厅中抢占主视觉：

- 花色选择
- 切牌
- 详情
- 高级设置

建议放入右上角：

```text
⋯
```

或：

```text
房间设置
```

# 9. 正式牌桌修改

## 当前问题

1. 木质桌边过厚；
2. 核心桌布区域过小；
3. 中央出牌区域偏小；
4. 玩家信息卡较小；
5. 底部手牌过于拥挤；
6. 多种信息重复表达。

---

## 9.1 缩小木质边框

建议：

- 桌面端木质桌边厚度减少约 25%；
- 移动端减少约 35%；
- 中央桌布区域扩大 15%–20%。

---

## 9.2 放大中央出牌区域

建议：

- 已出牌尺寸增加 20%–30%；
- 当前出牌人名称放在牌组上方；
- “不出”提示在中央停留 800ms 后淡出；
- 减少中央同时出现的光圈、环线和装饰。

---

## 9.3 优化底部手牌

当前“相同点数组成一列”的设计可以保留，但建议：

### 选中状态

```css
.card-column.selected {
  transform: translateY(-14px);
}

.card-column.selected::after {
  content: "";
  position: absolute;
  left: 10%;
  right: 10%;
  bottom: -6px;
  height: 3px;
  border-radius: 3px;
  background: var(--gold-primary);
}
```

### 数量提示

只保留一个数量徽章：

```text
右上角 ×3
```

不要同时出现：

- rank 圆点
- 数量文字
- 重复堆叠标签

---

## 9.4 当前玩家提示

建议：

- 当前玩家头像外圈发光；
- 非当前玩家保持静态；
- 不要让整个牌桌一起闪烁；
- 倒计时放在头像外圈或头像旁边。

---

## 9.5 操作按钮布局

移动端建议：

```text
[ 不出 ]   [ 提示 ]   [ 出牌 ]
```

其中：

- “出牌”为金色主按钮；
- “提示”为灰蓝按钮；
- “不出”为文字或透明按钮；
- “理牌”放在手牌区右上角，不和出牌按钮并列。

# 10. 移动端牌桌预览页修改

## 当前问题

存在：

```css
.table-preview {
  width: 100vw;
  min-width: 760px;
}
```

手机 390px 宽时页面实际横向溢出到约 760px。

## 修改

```css
.table-preview {
  width: 100%;
  max-width: 100vw;
  min-width: 0;
  overflow: hidden;
}
```

如果该页面只用于横屏展示，应：

1. 页面名称明确为“横屏牌桌预览”；
2. 竖屏时显示提示：

```text
请旋转手机查看牌桌预览
```

不要在竖屏中强行渲染 760px 宽内容。

const aiDifficulty = ref('medium')

const settings = storage.getSettings()

# 15. 移动端 AI 对局异常

实际运行中，移动端 AI 对局等待约 5.5 秒后仍停留：

```text
正在发牌……
```

控制台出现：

```text
finishDeal: invalid hand for selfSeat 0 st=false
```

桌面端同一路径可进入牌局。

## 建议排查

1. `selfSeat` 在移动端初始化时是否为 `0`；
2. `finishDeal()` 是否早于 snapshot / deal 完成；
3. 是否因移动端页面切换或监听顺序不同；
4. 是否存在重复初始化 game；
5. 是否在 `onMounted` 和路由监听中同时调用初始化；
6. 是否在手牌未写入时提前执行 `finishDeal()`。

## UI 兜底

即便逻辑异常，也不要永久停留在发牌状态。

建议：

```js
const DEAL_TIMEOUT = 8000

setTimeout(() => {
  if (phase.value === 'dealing') {
    showRetryDialog()
  }
}, DEAL_TIMEOUT)
```

弹窗：

```text
发牌超时

请检查房间连接后重试。

[ 重试 ]  [ 返回大厅 ]
```

# 17. 验收标准

## 移动端

- 390px 宽度下无横向滚动；
- 所有主要文字不小于 12px；
- 按钮高度不小于 44px；
- 页面左右安全边距不小于 16px；
- 房间号、IP 地址不异常换行；
- 设置页无控制台错误；
- AI 对局 8 秒内进入牌局或显示明确错误；
- 房间顶部不重叠；
- 底部按钮不被安全区遮挡。

## 桌面端

- 内容区不显得过小；
- 牌桌中心区域占主要面积；
- 木质桌边不超过页面视觉面积的 20%；
- 手牌和出牌区清晰可读；
- 页面最大内容宽度控制合理；
- 空白区域有节奏，不是单纯留空。

## 视觉统一

- 全项目只保留一套主色；
- 一级按钮数量每页不超过 1–2 个；
- 金色只用于强调；
- 不再使用原生 Emoji 作为主要功能图标；
- 面板圆角、阴影、字号统一；
- 首页、房间、牌桌切换时风格连续。

# 18. 最终建议

本轮最值得优先重做的是：

1. 房间大厅；
2. 正式牌桌比例；
3. 首页按钮层级；
4. 设置页信息结构。

当前项目已经有明确的棋牌游戏气质，下一步不需要继续增加更多发光、渐变和装饰，而应重点做：

> 减法、统一、层级和移动端可读性。

只要完成这一轮收敛，整体质感会比继续增加特效提升得更明显。
