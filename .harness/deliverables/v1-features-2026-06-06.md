## 2. 怎么手动验证

### 2.1 发牌动画

1. `npm run dev`（已在跑：http://localhost:8848/）
2. 浏览器打开 → 首页 → "开热点建房"
3. 房间页等 4 个标签都用同样房间号加入（同浏览器 4 标签可模拟）
4. 房主点"开局"
5. **看**：发牌时手牌从牌堆位置飞行到 4 个座位，每张牌 stagger ~80ms
6. 飞行中手牌区显示"发牌中"状态
7. 动画完成后可正常出牌

### 2.2 智能理牌按钮

1. 在游戏页（玩家手牌上方）找 💡"提示"按钮（橙黄渐变）
2. 点"提示" → AI 算推荐出牌，手牌中对应位置加金色高亮
3. 此时出现 ⚡"按提示出牌"绿色按钮（一级按钮变二级）—— 点它直接出推荐牌
4. 再点 💡 取消高亮
5. 没轮到玩家时按钮 disabled（灰色）

### 2.3 背景音乐 + 出牌声效

1. 首页底部"音效与音乐"卡片：
   - 勾掉"背景音乐"→ BGM 立即停
   - 拖动 BGM 音量滑块到 0 → BGM 静音
   - 勾掉"出牌声效" → 玩家 / AI 出牌时不发声
2. 设置自动保存到 localStorage，刷新页面记住
3. 进入对局后 BGM 自动启动（受开关控制）
4. 出牌时根据牌型听不同音：
   - 单张 / 对子：短促"啪"
   - 顺子 / 连对：上升琶音
   - 炸弹 4-7：低频"轰"
   - 炸弹 8+：多层"轰"
   - 王炸：电影级"轰！"

## 3. 已知小事项

- **音频首次播放需用户交互**：浏览器策略要求音频必须在用户手势后启动。HomeView 设置的 change 已经能触发；GameView 进入时也调用 `audio.unlock()` 解锁（页面跳转算用户交互）
- **没引入任何 CDN / 外部音频文件**：所有声音用 Web Audio API 实时合成（奥波音 / 三角波 / 噪音），符合"零公网依赖 + 离线"原则
- **发牌动画的 stagger 时长（80ms）和飞行时长（300ms）写在 `deal-animation.js` 顶部常量**，调参直接改
