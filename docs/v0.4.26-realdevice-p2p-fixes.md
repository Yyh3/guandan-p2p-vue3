> 日期：2026-07-20 ｜ 基线：v0.4.25（61 套件）→ 本次新增 v0426 套件（47 case）

## 一、用户反馈的 5 个 bug

| # | 现象 | 平台 |
|---|------|------|
| BUG-01 | 对局中改自己的昵称/头像，本人信息局中不刷新，只有局后才变 | 安卓 |
| BUG-02 | 加入房间页扫描局域网发现不了房间，只能输 IP+端口；希望能输房间号加入 | 安卓 |
| BUG-03 | 打开 App 不自动播放背景音乐 | 安卓 |
| BUG-04 | 网页端加入页和安卓不一样，输房间号进不去，也不能输 IP | 网页 |
| BUG-05 | 两台手机进同一房间，显示不同的房间号 | 安卓/跨设备 |

## 二、根因分析（对抗性审查）

### BUG-01：本人信息局中不刷新
- `RoomView.onNickConfirm` 旧版只更新**本地 reactive `peers`** + `net.broadcast({type:'NICK_UPDATE'})`。
- 但 `network.js` 内部的 `selfInfo` 和 `peers[selfSeat]` **没同步**；对局页 `useGameLogic.applyNetworkPlayers()` 读的是 `net.getPeers()` → 拿到旧昵称。
- 且 host 收到 joiner 的 `NICK_UPDATE` 后**不重广播 SYNC**，其它端要等下一次 JOIN/SYNC 才更新 → 表现为「局中不变、局后才变」。

### BUG-02：局域网扫描发现不了房间
- `scanLanRooms` 的 `COMMON_SUBNETS` 只猜固定网段，**缺 `192.168.137`（Windows 手机热点常用网段）**，热点网段不在列表就扫不到。
- 没有「按本机 IP 推导本子网」的能力；也没有「输房间号自动匹配加入」的入口。

### BUG-03：不自动播 BGM
- Android WebView 默认 `MediaPlaybackRequiresUserGesture = true`，`audio.startBgm()` 的 `<audio>.play()` 被静默拒绝。

### BUG-04：网页端加入页缺 IP 入口
- 网页端 `JoinView` 只有房间号输入（走 BroadcastChannel，仅同机多标签），**没有 IP:端口输入框**，无法跨设备加入真实 WS host。

### BUG-05：两端房间号不一致
- host 权威 `SYNC` payload **不带 `roomNo`**；joiner 直连（仅输 IP、URL 无 roomNo）时 `pickRoomNo()` **随机生成**一个房号 → 与 host 显示的不同。

## 三、修复方案

### Fix A — `src/common/network.js`
- 新增 `updateSelfInfo(patch)`：统一更新 `selfInfo` + `peers[selfSeat]`，广播 `NICK_UPDATE`，host 额外重广播 `SYNC`。命名导出 + 默认 `net` 对象都导出。
- 4 处 SYNC payload（reuse/join 定向 + 两处广播）统一带上 `roomNo: roomId`。
- `NICK_UPDATE` host handler 收到后重广播 `SYNC`（带 roomNo）。
- `COMMON_SUBNETS` 补 `192.168.137`。
- 新增 `buildScanCandidates(localIp, port)`：按本机 IP 推导本子网候选（网关 `.1` 优先、跳过自己、扇扫 `.2..60`），再接默认候选。命名导出 + 默认对象都导出。

### Fix B — `src/views/room/RoomView.vue`
- `message:SYNC` handler：joiner（非 host）从 `payload.roomNo` 同步 host 真实房号（`roomNo.value` + `net.setRoomId`）。
- `onNickConfirm` 改走 `net.updateSelfInfo({nickname, avatar})`（保留 `broadcast` 兜底）。

### Fix C + D — `src/views/join/JoinView.vue`
- 浏览器分支补 IP:端口输入框（`v-model="hostAddress"`）+ 房间号输入；`addressError` 去掉平台门控；`onJoin` 浏览器手输 IP 走 `role=joiner&host=` 跨设备加入。
- native 分支补「或房间号」输入；`onJoin` 改 `async`，仅输房间号时先 `await scanRooms()` 自动按房间号匹配回填 `hostAddress`。
- `scanRooms` native 用 `net.buildScanCandidates(localIp)`（`localIp !== '127.0.0.1'` 守卫），扫到后按 `roomNo` 匹配 `selectRoom`。
- 合并导入 `import WsServer, { isNativeCapacitor } from '@/common/ws-server.js'`。

### Fix E — `android/.../MainActivity.java`
- `bridge.getWebView().getSettings().setMediaPlaybackRequiresUserGesture(false)`（try/catch 兜底，拿不到 WebView 时退回 main.js 的 pointerdown 解锁）。
