# UI 美化与用户留存建议（v0.4.27）

> 基于对抗性审查同期的 UI/HCI 调研产出。本文档只给建议与优先级，不改动代码。
> 所有建议均立足于项目**已有的翡翠绿 + 金色玻璃拟态主题**（`tokens.css` v3.x 段），
> 不推翻现有设计语言，只做"在既有身份上的增量打磨"。

---

## 一、现有设计资产盘点（先肯定，再增量）

项目已经有一套**成熟且有辨识度**的视觉体系，这是留存的地基，不要推倒：

| 资产 | 位置 | 评价 |
|------|------|------|
| 翡翠绿径向牌桌 + 金色装饰 | `tokens.css` `--felt-base` / `--gold-metallic` | 掼蛋=传统纸牌，翡翠绿毡+金箔是"高级棋牌室"的正确气质 |
| 奶油白牌面 + 金边 + 卷草纹 | `--card-cream` / `--card-pattern-svg` | 牌面是卡牌游戏的核心资产，已做出"实体扑克"质感 |
| 木边椭圆牌桌 + 顶部聚光 | `app-theme.css` `.app-half-table-bg` | 俯视角牌桌隐喻，空间感强 |
| 身份三色（队友蓝/对手红/自己绿） | `--color-teammate/opponent/self` | 4 人游戏的关键信息分层，做得对 |
| 完整 token 体系 + 简洁模式降级 | `tokens.css` / `simple-mode.css` | 工程化程度高，任何新装饰都能安全降级 |
| 发牌动画 / 炸弹特效 / 飞牌轨迹 | `deal-animation.js` / `EffectLayer.vue` | 动效基础好，"活"的底子已有 |

**核心结论**：视觉语言不缺，缺的是**把"好看"转化为"想回来"**的留存闭环。
下面按"投入产出比"分三档给建议。

---

## 二、P0 快速见效（小改动，直接影响"第一次/下一次还玩不玩"）

### P0-1 首启引导：把技术黑话换成"30 秒开局"图卡

**现状问题**：首页 `capability-hint` 是一句长技术文案
（"当前为浏览器,创建的房间仅支持本机多标签联机;跨手机请用 Android App"）。
第一次打开的人看不懂"多标签联机"，**开局门槛 = 流失**。

**建议**：
- 首次启动弹一张**一次性**图卡（localStorage 记 `seenOnboarding`）：
  「① 一人开热点 → ② 朋友连热点扫码/输 IP → ③ 开打」，配 3 个简笔图标。
- 技术文案降级为图卡底部小字"了解更多"。
- 留存逻辑：**第一次 60 秒内开不了局的人，基本不会回来**。这是 ROI 最高的一项。

### P0-2 首页增加"回到上次的牌局"

**现状问题**：首页主 CTA 永远是"开始游戏"（建房）。但回访用户最高频的动作是
**找回上次那桌人**，现在要重新走一遍建房/加入流程。

**建议**：
- localStorage 记最后一局（房间号/host IP/时间/牌局是否还在）。
- 首页若有"未散的局"或"24h 内的局"，主 CTA 上方浮一条金色横幅
  「♠ 回到上次的牌局 · 房间 8848」，一点直达。
- 留存逻辑：**减少回访摩擦 = 直接提升次日留存**。

### P0-3 结算页做一张"值得截图发群"的战绩卡

**现状问题**：结算信息齐全但偏"报表"，没有仪式感，也没有可传播的形态。

**建议**：
- 结算时生成一张**金箔"胜/负"印章 + 双方比分 + 本局亮点**（如"双响炮×2"）的卡片。
- 提供"保存战绩图"（canvas 导出）。**离线场景下，截图发微信群就是唯一的自传播**——
  朋友看到一张精致的战绩图，比任何文案都拉新。
- 亮点词用掼蛋黑话（"春天""光头""炸弹王"），强化圈子认同。

### P0-4 微交互补光：让"能出的牌"自己发光

**现状问题**：`card-hint-shine` 金光闪 token 已存在，但只在个别路径用。

**建议**：
- 轮到自己且手牌里有能压过的牌时，**整列可出牌**统一走 `card-hint-shine` 呼吸金光。
- 桌面端手牌 hover 加 3D 轻微抬升（`translateY(-4px) rotateX`），出牌落桌加"啪"一声吸附。
- 留存逻辑：**"轮到我了/我能出"的即时视觉反馈**是卡牌游戏爽感的最小单元，成本低、体感提升大。

## 三、P1 中期建设（进度感 = 离线留存的发动机）

> 本项目**无服务器**，所以所有进度必须落 localStorage。
> 离线留存的本质：**给用户一个"我在这副牌里攒了东西"的理由**。

### P1-1 本地成就 / 称号系统

- 用掼蛋术语做成就：「炸弹王」（单局≥3 炸）、「春天使者」（打出春天）、
  「铁树开花」（同花顺绝杀）、「常胜将军」（5 连胜）。
- 个人面板展示徽章墙（金色徽章 = 已得，灰 = 未得）。
- 成就达成时牌桌中央弹金色徽章动画（复用 `EffectLayer` 的 bomb-pop 通道）。
- 留存逻辑：**收集欲是最强的离线回访动机**，且全部本地实现、零联网。

### P1-2 个人数据面板（把 HistoryView 从"流水账"升级为"我的牌风"）

- 现有 `HistoryView` 是记录列表。增加聚合层：
  胜率、场均炸弹数、最爱牌型、最长连胜、**牌风雷达图**（进攻/防守/运气/牌运四维）。
- 留存逻辑：玩家爱看自己的数字，"我胜率 62%"是会挂在嘴边的谈资。

### P1-3 可解锁的牌背 / 桌布主题

- 现有 `--card-back-svg` 深红牌背 + 翡翠桌布是默认款。
- 增加 2~3 套可解锁主题（成就解锁，非付费）：墨绿竹纹、绛红漆器、月白青花。
- 设置页"牌桌"分区预览切换。
- 留存逻辑：**自定义 = 拥有感**。解锁制又和 P1-1 成就咬合成进度闭环。

### P1-4 AI 生涯模式（给单机玩家一个爬梯）

- 现有 AI 只有 medium/hard 两档，打完就结束，**单机没有进度**。
- 建议加"生涯"：从 2 打到 A，每级 AI 难度递增 + 给 AI 起名字/性格
  （"稳重的老周""爱炸的小李"），升级时金色"晋级"动画。
- 留存逻辑：**高铁/隧道场景大量是单人 + 3AI**，单机进度是这部分用户唯一的留存抓手。

## 四、P2 长期打磨（质感与氛围，锦上添花）

### P2-1 字体升级：给"掼蛋"和牌面数字一张有性格的脸

- 现状全站 `system-ui / PingFang SC`，干净但没有记忆点。
- 建议：**标题/牌面点数/分数**用一款有"纸牌/书法"气质的展示字体
  （如思源宋体 Heavy 或免费商用的演示佛系体），正文仍用 PingFang。
- 「展示字体 + 正文字体」的配对是"精致感"最便宜的提升。
- 注意：离线项目**不用 CDN 字体**，需把 woff2 子集化后打进 `assets/`（只打包用到的字符）。

### P2-2 动态牌桌氛围（克制的"活"）

- 木边加缓慢流动的光泽反射；顶光束里加极淡的浮尘粒子；
  手牌阴影随"抬牌高度"变化。
- **必须全部挂 simple-mode 开关**（项目已有降级机制，照 `simple-mode.css` 模式加）。
- 留存逻辑：环境"活"但"不吵"，是高级感和廉价感的分水岭。

### P2-3 声音景观分层

- 现有 BGM + 按钮音 + 炸弹音。建议补：
  房间环境底噪（极轻的纸牌窸窣）、**剩牌少时音乐张力升高**、胜利号角短句。
- 音频是"沉浸感"里性价比最高的一层，且纯本地合成（WebAudio）零资源。

### P2-4 平板横屏一等公民

- 4 人牌局经常在**一台 iPad 上传着玩**。确保横屏 4 人布局在平板上是
  "围桌而坐"的完整形态（已有横屏分支，建议作为独立测试目标持续打磨）。

## 五、必须守住的设计原则（避免改坏）

1. **翡翠绿 + 金是身份色**，任何新元素先问"它配不配这套色"，不引入靛蓝/紫粉渐变等通用 App 色。
2. **一切装饰必须能被 simple-mode 关掉**——新增动效/氛围时同步在 `simple-mode.css` 加降级。
3. **离线优先**：所有进度/成就/主题只存 localStorage，绝不引入联网依赖（项目红线）。
4. **牌面是圣域**：牌面尺寸/配色/可读性已被大量测试锁定（`v0425` 系列），改牌面必须全量回归。
5. **先截图核对再合**：视觉改动用 Playwright 截图人工比对（项目已有 `review-screenshots.mjs` 流程）。

## 六、落地优先级一览

| 优先级 | 项目 | 留存价值 | 工作量 |
|--------|------|----------|--------|
| P0-1 | 首启引导图卡 | ★★★★★ | 小 |
| P0-2 | 回到上次的牌局 | ★★★★★ | 小 |
| P0-3 | 可截图战绩卡 | ★★★★☆ | 中 |
| P0-4 | 可出牌金光/hover | ★★★★☆ | 小 |
| P1-1 | 本地成就系统 | ★★★★★ | 中 |
| P1-2 | 数据面板 | ★★★★☆ | 中 |
| P1-3 | 可解锁主题 | ★★★☆☆ | 中 |
| P1-4 | AI 生涯模式 | ★★★★☆ | 大 |
| P2-* | 字体/氛围/音效/平板 | ★★★☆☆ | 中~大 |

**建议执行顺序**：P0-1 → P0-2 → P0-4（三个小改动先收割留存），
再做 P1-1 + P1-2（进度闭环），其余按版本节奏排期。
